## 📖 学习内容

### 1. LLM 推理与决策
- StepFun Step-3.5 技术报告
- OpenAI GPT-4o 系统卡
- Anthropic Claude 3.5 安全研究
- Google Gemini 1.5 Pro 长上下文应用

**关键发现**:
- Step-3.5-flash 在速度和质量间取得良好平衡，适合高频交易
- 128K 上下文足够覆盖近期市场数据和策略历史
- 成本控制: 输出 token 比输入贵 2x，需优化 prompt 长度

### 2. RAG (检索增强生成)
- LangChain 文档与最佳实践
- LlamaIndex 高级用法
- Milvus/Pinecone 向量数据库选型
- Embedding 模型对比 (text-embedding-ada-002, bge-m3)

**实施方案**:
```
用户查询 → 向量化 → Milvus检索 (top-5) → 拼接上下文 → LLM生成 → 返回结果
```

**性能指标**:
- 检索延迟: <10ms (缓存命中)
- 生成延迟: <200ms (Step-3.5-flash)
- 总延迟: <300ms

### 3. Agent 自进化
- DeepMind 的 Agent 元学习
- OpenAI 的 Emergent Abilities
- AutoGPT 的自我反思机制
- BabyAGI 的任务分解与优先级

**设计思路**:
- 每个 Agent 维护自己的 "经验库" (成功/失败案例)
- 定期运行 "反思会议" (多个 Agent 讨论策略)
- 进化算法: 调整参数 → 回测 → 评分 → 保留最优

### 4. 高性能计算
- Node.js 集群模式 (Cluster Mode)
- Python asyncio + uvloop
- ZeroMQ 用于进程间通信
- Redis Streams 作为事件总线

**基准测试**:
| 技术栈 | 吞吐量 (msg/s) | 延迟 (ms) | CPU 占用 |
|--------|----------------|-----------|----------|
| Node.js 单进程 | 8000 | 15 | 60% |
| Node.js 集群 (4核) | 25000 | 12 | 85% |
| Python asyncio | 12000 | 25 | 70% |
| 混合模式 (推荐) | 30000+ | <10 | 90% |

### 5. 数据管道
- Apache Kafka vs Redis Streams
- 数据序列化 (Avro vs JSON)
- Schema 演化与兼容性
- 死信队列处理

**选择理由**:
- Redis Streams 更轻量，适合单节点部署
- Avro 节省带宽，但增加序列化开销
- 当前规模 (1W msg/s) Redis 足够

## 📈 性能优化策略

### 1. 减少序列化开销
- 使用二进制协议 (MessagePack/Protobuf)
- 复用对象，避免频繁 GC
- 预分配内存池

### 2. 提高并发
- Node.js 集群 + 共享端口
- Redis Pipeline 批量操作
- 数据库连接池调优 (max 50)

### 3. 降低延迟
- 关键路径避免同步 I/O
- 使用内存缓存 (LRU)
- 预热连接池

### 4. 监控热点
- 使用 0x 或 clout  profiling
- 追踪长尾延迟 (p95, p99)
- 设置自动告警

## 二、重点模块详细部署步骤

### 2.1 OKX Agent Trade Kit 部署

#### 前置条件
- ✅ Node.js 18+
- ✅ npm 可访问（当前网络受限）
- ⚠️ OKX API Key（可选，行情查询无需）

#### 步骤1: 全局安装MCP服务器
```bash
npm install -g @okx_ai/okx-trade-mcp

# 验证安装
okx-trade-mcp --help
```

#### 步骤2: 安装OpenClaw Skills
```bash
npx skills add okx/agent-skills

# 如果失败（网络限制），手动下载：
# 1. 访问 https://clawhub.com/okx/agent-skills
# 2. 下载zip并解压到 ~/.openclaw/workspace/skills/okx-agent-skills
```

#### 步骤3: 配置MCP服务器
创建 `~/.openclaw/openclaw.json` 添加：

```json
{
  "skills": {
    "entries": {
      "okx-cex-market": {
        "command": "npx",
        "args": ["@okx_ai/okx-trade-mcp", "--modules", "market"],
        "env": {
          "OKX_API_KEY": "your_api_key",
          "OKX_SECRET_KEY": "your_secret",
          "OKX_PASSPHRASE": "your_passphrase"
        }
      },
      "okx-cex-trade": {
        "command": "npx",
        "args": ["@okx_ai/okx-trade-mcp", "--modules", "swap"],
        "env": {
          "OKX_API_KEY": "your_api_key",
          "OKX_SECRET_KEY": "your_secret",
          "OKX_PASSPHRASE": "your_passphrase"
        }
      }
    }
  }
}
```

#### 步骤4: 启动MCP并测试
```bash
# 启动MCP服务器（后台）
okx-trade-mcp --profile live --modules market,swap &
sleep 2

# 测试行情查询（无需API Key）
okx market ticker BTC-USDT

# 测试交易（需要API Key）
okx swap place_order BTC-USDT 1 BUY market
```

#### 注意事项
- 🔐 **安全**: API Key存储在 `~/.okx/config.toml`，不进入对话历史
- 🧪 **测试**: 先用demo模式，`okx-trade-mcp --profile demo`
- 📊 **监控**: 使用 `okx account get_balance` 检查余额

### 2.2 OpenSpace 自我进化系统部署

#### 前置条件
- ✅ Python 3.12+ (当前: 3.12.3 ✅)
- ✅ git 可用

#### 步骤1: 克隆仓库
```bash
cd /tmp
git clone https://github.com/HKUDS/OpenSpace.git
cd OpenSpace

# 如果github.com访问失败，使用镜像：
# git clone https://hub.nuaa.cf/HKUDS/OpenSpace.git
```

#### 步骤2: 安装Python包
```bash
pip install -e . --no-cache-dir

# 验证安装
python3 -c "import openspace; print('OpenSpace version:', openspace.__version__)"
```

#### 步骤3: 配置MCP
```bash
mcporter config add openspace \
  --command "openspace-mcp" \
  --transport stdio
```

#### 步骤4: 复制Skills到OpenClaw
```bash
# OpenSpace自带host_skills，复制到OpenClaw技能目录
cp -r skills/* ~/.openclaw/workspace/skills/

# 主要技能:
# - delegate-task: 任务委派
# - skill-discovery: 技能发现
```

#### 步骤5: 配置环境变量
```bash
export OPENSPACE_HOST_SKILL_DIRS="$HOME/.openclaw/workspace/skills"
export OPENSPACE_WORKSPACE="/tmp/OpenSpace"
# 可选: export OPENSPACE_API_KEY="sk-xxx"

# 持久化到 ~/.bashrc 或 ~/.profile
```

#### 步骤6: 启动OpenSpace MCP
```bash
# 方法A: 通过mcporter启动（推荐）
mcporter run openspace

# 方法B: 直接启动
openspace-mcp --host-skill-dirs "$OPENSPACE_HOST_SKILL_DIRS" &
```


## 三、系统整合优化

### 3.1 事件总线桥接

**目标**: 将 event-bus-v2 的零延迟能力整合到 OpenClaw 核心

**方法**: 修改 `scripts/integration-orchestrator.js`

```javascript
// 已有: communication-bridge.js
// 新增: 桥接event-bus-v2
const { ZeroLatencyEventBus } = require('../multi-agent-system/event-bus-v2');

class IntegrationOrchestrator {
  constructor() {
    this.eventBus = new ZeroLatencyEventBus();
    this.initializeBridges();
  }
  
  initializeBridges() {
    // 桥接OpenClaw事件到ZeroLatencyBus
    this.bindOpenClawEvents();
    // 桥接OpenSpace进化事件
    this.bindOpenSpaceEvents();
  }
  
  bindOpenClawEvents() {
    // 任务开始 → 零延迟广播
    this.on('task:start', (event) => {
      this.eventBus.publish('task:start', event, 'openclaw', { priority: 1 });
    });
    
    // 任务进度 → 缓冲合并
    this.on('task:progress', (event) => {
      this.eventBus.publish('task:progress', event, 'openclaw', { priority: 3 });
    });
    
    // 任务完成 → 立即处理
    this.on('task:complete', (event) => {
      this.eventBus.publish('task:complete', event, 'openclaw', { priority: 0 });
    });
  }
}
```
